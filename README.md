# null-key

**Детерминированный генератор паролей — один секрет вместо менеджера паролей.**

Уникальный пароль для каждого сервиса выводится на лету из его названия и вашей секретной фразы. Ничего не хранится и не уходит из браузера — пароль можно воспроизвести в любой момент, зная те же исходные данные.

## О приложении

**null-key** — это альтернатива хранилищу паролей. Вместо того чтобы где-то держать базу с вашими паролями,
он **вычисляет** их заново каждый раз из двух значений: названия сервиса (например, `github`) и одной секретной
фразы, которую помните только вы. Одинаковые исходные данные всегда дают один и тот же пароль, поэтому хранить
сами пароли не нужно.

Вся генерация происходит локально в браузере через **Web Crypto API**. Нет сервера, нет базы, нет `localStorage` —
секретная фраза и готовый пароль никуда не передаются и нигде не сохраняются.

> Сервис заточен под использование **на телефоне**. Его удобно добавить на домашний экран (особенно на iOS) и пользоваться как обычным приложением — без адресной строки и в полный экран. См. раздел [«Установка на телефон»](#установка-на-телефон-как-приложение).

## Возможности

- **Детерминированная генерация** — одни и те же название сервиса, секретная фраза и длина всегда дают один и тот же пароль.
- **Без хранения** — пароли нигде не сохраняются, их не нужно где-то держать: достаточно помнить одну секретную фразу.
- **Полностью офлайн** — вся криптография выполняется в браузере через Web Crypto API, ни один байт не уходит в сеть.
- **Настраиваемая длина** — ползунок от 8 до 48 символов; в наборе буквы обоих регистров, цифры и спецсимволы.
- **Показать/скрыть фразу** — секретная фраза по умолчанию замаскирована.
- **Копирование в один тап** — готовый пароль копируется в буфер обмена.
- **Плавные анимации** — на базе Framer Motion, с учётом системной настройки «уменьшить движение».

## Как это работает

Пароль выводится по детерминированной схеме на основе HMAC и SHA‑256:

1. `HMAC-SHA256(секретная фраза, название сервиса)` — даёт стартовое состояние, привязанное к конкретному сервису.
2. Состояние **растягивается** повторным `SHA-256`, пока не наберётся нужное количество символов.
3. Байты состояния отображаются в набор символов (буквы верхнего и нижнего регистра, цифры, спецсимволы).

Поскольку функция чистая и детерминированная, один и тот же ввод (с точностью до UTF‑8) всегда воспроизводит тот же
пароль. Реализация — в [`src/lib/derivePassword.ts`](src/lib/derivePassword.ts).

> ⚠️ Стойкость итогового пароля определяется стойкостью вашей секретной фразы. Выбирайте длинную и неочевидную фразу
> и не используйте одну и ту же на ненадёжных устройствах.

## Технологии

- **React 19** + **TypeScript**
- **Vite 8** — сборка и dev‑сервер
- **Tailwind CSS 4** — стилизация и дизайн‑токены (палитра в `src/index.css`)
- **Framer Motion 12** — анимации (с поддержкой `prefers-reduced-motion`)
- **lucide-react** — иконки
- **Web Crypto API** (`crypto.subtle`) — HMAC‑SHA256 и SHA‑256

## Запуск

Нужен **Node.js 18+** и любой пакетный менеджер (примеры на `npm`).

```bash
# установка зависимостей
npm install

# запуск dev-сервера (по умолчанию http://localhost:5173)
npm run dev

# сборка production-версии в dist/
npm run build

# локальный предпросмотр собранной версии
npm run preview

# линтинг
npm run lint
```

## Установка на телефон как приложение

null-key — это статический сайт (PWA), поэтому его можно «установить» прямо из браузера и запускать в
полноэкранном режиме, как нативное приложение.

**iOS (Safari):**
1. Откройте сайт в Safari.
2. Нажмите кнопку «Поделиться» (квадрат со стрелкой вверх).
3. Выберите **«На экран Домой»** → **«Добавить»**.
4. Запускайте null-key с домашнего экрана — он откроется без адресной строки.

**Android (Chrome):**
1. Откройте сайт в Chrome.
2. Меню (⋮) → **«Добавить на главный экран»**.

После сборки (`npm run build`) папку `dist/` можно развернуть на любом статическом хостинге (GitHub Pages, Netlify, Vercel, Cloudflare Pages) и открыть с телефона.

## Приватность

- Секретная фраза и сгенерированный пароль **никуда не отправляются** и **нигде не сохраняются** — ни на сервере, ни в `localStorage`.
- Вся работа происходит в памяти вкладки; после закрытия не остаётся ни паролей, ни истории.
- Приложение работает полностью офлайн.
